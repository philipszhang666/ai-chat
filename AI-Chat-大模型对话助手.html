📋 计划模式

💡 流程：① AI 生成执行计划（工具开启时可先侦察项目）→ ② 老师评审计划 → ③ 规划者根据意见修订 → ④ 按计划逐步执行 → ⑤ 整合输出 → ⑥ 独立老师验证结果。
适合：复杂任务（写报告、研究、多步推理、代码项目等）。
⚠️ 调用 LLM 多次，成本较高。
让老师审查并改进计划（推荐）
把步骤结果整合（关闭则直接拼接）
执行完成后由独立老师基于任务、方案和结果验证，可自主调用工具

2
2
每轮未通过后，需要你手动决定是否追加改进阶段；不会自动重跑原步骤。

📑 大纲模式 - 动态规划

💡 机制：AI 在单一上下文中边做边维护一份"工作大纲"，发现新需求会动态追加条目，全程使用工具。
与计划模式的区别：计划模式是"先规划→人审批→分步执行"的瀑布式；大纲是"自主探索 + 动态调整"的敏捷式。
适合：探索性任务、代码调试、需要边做边发现的工作。
⚠️ 强依赖工具调用能力，请确保已加载至少一个工具。
启用后会自动关闭计划模式 / 师生模式
每次 LLM 调用算一轮。受您的请求频率限制保护，超限自动等待。
大纲模式需要良好的 function calling 能力，建议用 GPT-4o / Claude Sonnet / DeepSeek-Chat 等
指导 AI 如何使用 save_outline / append_outline / update_outline 三个工具维护大纲。
